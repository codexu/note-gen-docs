Skill 是包含名称、描述和 Markdown 指令的能力包。Agent 根据描述自动匹配 Skill，也可以通过斜杠菜单手动调用允许用户调用的 Skill。

## 全局与工作区 Skills

| 类型 | 位置与范围 |
| --- | --- |
| 全局 Skills | 保存在 NoteGen 的全局 Skill 目录，可用于所有工作区 |
| 工作区 Skills | 保存在当前工作区的 `skills` 目录，只对该工作区生效 |

工作区 Skills 随工作区文件一起备份或同步。它们不会作为普通知识库文档计算向量。

## 创建和编辑

一个 Skill 至少包含 `SKILL.md`。在设置中创建时可填写：

- 名称与描述，描述用于自动匹配；
- 版本与作者；
- 是否显示在斜杠菜单；
- 允许工具，列出的工具可按 Skill 规则减少重复确认；
- Markdown 指令内容。

指令应说明适用场景、输入、步骤、输出格式和安全边界，避免把 API Key 等密钥写入 Skill。

## 导入与导出

可导入包含 `SKILL.md` 的 ZIP，也可导出全局 Skill 以便分享。导入前检查指令和允许工具；Skill 本身不会自动安装未声明的外部程序。

## 自动匹配

启用 Skills 和自动匹配后，Agent 会根据用户输入与 Skill 描述选择候选能力。描述越具体，误匹配越少。自动匹配不理想时，可关闭它并通过 `/skill-name` 手动触发。

Agent Skills 是开放格式。不同产品对元数据、工具权限和运行时支持可能不同，跨产品导入后应重新检查。
