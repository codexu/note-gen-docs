### 启用图床

开启图床功能后，上传的图片将自动保存到配置的图床服务。

### 主要图床

选择 GitHub、SM.MS、PicGo 或 S3 作为主要图床，设置后才可生效。

### GitHub 图床

使用 GitHub 仓库存储图片。

#### 配置项

| 配置项 | 说明 |
|-------|------|
| 仓库 | 用于存储图片的 GitHub 仓库 |
| 分支 | 仓库分支名称 |
| Token | GitHub Personal Access Token |

#### jsDelivr 加速

开启后，图片链接将使用 jsDelivr CDN 加速，提升国内访问速度。

> 注意：GitHub 图床需要创建公开仓库，否则图片将无法正常访问。

### SM.MS 图床

使用 SM.MS 图床服务存储图片。

#### 配置项

| 配置项 | 说明 |
|-------|------|
| API Key | SM.MS API 密钥 |

#### 磁盘使用

显示当前 SM.MS 账号的存储空间使用情况。

### PicGo 图床

通过 PicGo 服务上传图片，需要先启动 PicGo 应用。

#### 配置项

| 配置项 | 说明 |
|-------|------|
| 服务地址 | PicGo 服务地址，默认为 http://127.0.0.1:36677/upload |

### S3 图床

使用 AWS S3 或兼容 S3 API 的存储服务。

#### 配置项

| 配置项 | 说明 |
|-------|------|
| Access Key | 访问密钥 ID |
| Secret Key | 访问密钥 |
| Bucket | 存储桶名称 |
| Region | 存储桶所在区域 |
| 自定义域名 | 可选的 CDN 加速域名 |

#### 兼容服务

S3 图床支持以下服务：
- AWS S3
- 阿里云 OSS
- 腾讯云 COS
- 七牛云
- MinIO
- 其他兼容 S3 API 的服务
