### 启用图床

开启图床功能后，上传的图片将自动保存到配置的图床服务。

### 主要图床

选择 GitHub、SM.MS、PicGo 或 S3 作为主要图床，设置后才可生效。

### GitHub 图床

使用 GitHub 仓库存储图片。

#### 配置项

| 配置项 | 说明 |
|-------|------|
| 仓库 | 用于存储图片的 GitHub 仓库 |
| 分支 | 仓库分支名称 |
| Token | GitHub Personal Access Token |

#### jsDelivr 加速

开启后，图片链接将使用 jsDelivr CDN 加速，提升国内访问速度。

> 注意：GitHub 图床需要创建公开仓库，否则图片将无法正常访问。

<a id="token-permissions-guide" />

#### Access Token 权限

推荐使用 GitHub Fine-grained personal access token：

- 图床仓库已存在：Repository access 只选择图床仓库，并设置 `Contents: Read and write`。
- 由 NoteGen 创建图床仓库：Repository access 选择 `All repositories`，并设置 `Contents: Read and write` 和 `Administration: Read and write`。创建完成后，建议换成只授权图床仓库的新令牌。
- `Metadata: Read` 会由 GitHub 自动包含，无需额外设置。

- [创建 GitHub Fine-grained personal access token](https://github.com/settings/personal-access-tokens/new?name=NoteGen&description=NoteGen+image+hosting&expires_in=none&contents=write&administration=write)：预填长期有效期，以及 Contents 和 Administration 权限。

GitHub 仍会要求你确认资源所有者和仓库范围。

### SM.MS 图床

使用 SM.MS 图床服务存储图片。

#### 配置项

| 配置项 | 说明 |
|-------|------|
| API Key | SM.MS API 密钥 |

#### 磁盘使用

显示当前 SM.MS 账号的存储空间使用情况。

### PicGo 图床

通过 PicGo 服务上传图片，需要先启动 PicGo 应用。

#### 配置项

| 配置项 | 说明 |
|-------|------|
| 服务地址 | PicGo 服务地址，默认为 http://127.0.0.1:36677/upload |

### S3 图床

使用 AWS S3 或兼容 S3 API 的存储服务。

#### 配置项

| 配置项 | 说明 |
|-------|------|
| Access Key | 访问密钥 ID |
| Secret Key | 访问密钥 |
| Bucket | 存储桶名称 |
| Region | 存储桶所在区域 |
| Endpoint | S3 兼容服务地址，AWS 可使用默认值 |
| 自定义域名 | 可选的 CDN 加速域名 |
| 路径前缀 | 存储桶内用于组织图片的目录前缀 |

#### 兼容服务

S3 图床支持以下服务：
- AWS S3
- 阿里云 OSS
- 腾讯云 COS
- 七牛云
- MinIO
- 其他兼容 S3 API 的服务
