NoteGen separates provider configuration from scenario models. First add providers and models under [Model Configuration](/en/docs/settings/model-config), then select models for individual features.

## Fallback behavior

The primary chat model is the default fallback for conversational features. A scenario normally uses it when no model is selected. Choosing No Model disables the corresponding AI capability.

## Where to configure models

| Scenario | Settings page | Recommendation |
| --- | --- | --- |
| Chat / Agent | Chat | Strong chat and tool-calling model |
| Conversation condensation | Chat | Low-cost model that summarizes well |
| Editor completion | Editor | Low-latency chat model |
| Record descriptions | Record or Default Model | Small chat model |
| Knowledge-base embedding/rerank | Knowledge Base | Match the embedding/rerank type |
| Image understanding | Image Recognition | Vision-capable model |
| Speech to text | Audio | STT model |
| Read aloud | Read Aloud | TTS model or system voice |

After changing the embedding model, rebuild knowledge-base vectors so vectors from different models are not mixed.
