Local backup and cloud sync serve different purposes. Sync keeps devices updated; a local ZIP protects migrations, risky operations, and offline archives.

<img src="https://files.seeusercontent.com/2026/07/16/Eka3/desktop-backup.jpeg" alt="Desktop backup and sync menu" className="rounded-xl border" />

## Export

1. Open the sync-status menu in the title bar.
2. Choose Export under Local Backup (All).
3. Select a destination.

The archive includes application databases, configuration, and relevant files from the default workspace. If you use a custom workspace, also copy the full workspace directory so external assets are covered.

## Import

1. Choose Import from the same menu.
2. Select a ZIP exported by NoteGen.
3. Confirm the restore and allow NoteGen to restart.

Import overwrites current application data. Export the current state first, do not alter the ZIP structure, and do not import untrusted archives.

## Recommended strategy

- Back up before major upgrades, workspace migrations, and sync-conflict overwrites.
- Cloud sync is not an offline backup; remote deletion may propagate to other devices.
- Periodically verify that archives can be opened and keep at least one copy on a separate device.
