MCP（Model Context Protocol）让 NoteGen 通过统一协议调用外部工具和资源。启用 MCP 不会自动授权所有服务器；还需要启用服务器并在对话中选择它。

## 连接类型

### 本地命令（stdio）

仅支持桌面端。填写命令、空格分隔的参数和 JSON 格式环境变量，例如：

```json
{
  "API_KEY": "your-key"
}
```

命令字段可直接填写 `npx @modelcontextprotocol/server-filesystem`，NoteGen 会在参数字段为空时拆分第一个命令和后续参数。

### HTTP 服务

桌面端和移动端都支持。填写服务 URL，以及可选的 JSON 请求头：

```json
{
  "Authorization": "Bearer your-token"
}
```

移动端不能启动本机 stdio 进程，因此新建服务器时只提供 HTTP 类型。

## 添加与导入

- 点击“添加服务器”，填写名称、类型和连接参数。
- 可关闭单个服务器而不删除配置。
- “导入 JSON”支持常见的 `mcpServers` 配置结构；重复服务器会被跳过。
- 保存前可测试临时配置，保存后启用的服务器会自动尝试连接。

## 运行时环境

本地命令常依赖 Node.js、Python、uv 或其他启动器。使用“检查环境”查看版本和缺失项；支持的平台可以在确认命令后执行引导式安装。安装输出会显示在日志中，也可以中途终止。

## 工具浏览与使用

连接成功后，工具浏览器会显示服务器提供的工具、描述和参数。打开 Chat/Agent 工具栏中的 MCP 按钮，选择要用于当前会话的服务器。Agent 调用工具时仍会遵循权限确认规则。

## 安全

- 本地命令能以当前用户权限运行程序，只使用可信配置。
- 环境变量和请求头可能包含密钥，不要截图或公开导出配置。
- 只给远程 MCP 发送完成任务所需的最少上下文。
- 删除服务器不可撤销，但不会自动撤销第三方服务中的 Token。
