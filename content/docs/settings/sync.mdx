### 主要同步方式

主要同步方式目前支持 GitHub、Gitee、GitLab、Gitea，原因是历史回滚功能需要 Git（本地无需安装 Git） 的支持。

#### 配置

##### GitHub/Gitee
- GitHub Access Token，需要去 [GitHub](https://github.com/settings/tokens/new) 或 [Gitee](https://gitee.com/profile/personal_access_tokens) 获取令牌，注意需要访问仓库的权限。
- 自动同步，开启后，在写作时，停止输入后的 * 秒后进行自动快速备份，提交内容为固定模板，可折叠。

##### GitLab/Gitea
- 支持 GitLab 和 Gitea 自建实例，需要配置服务器地址和访问令牌。
- 同样支持历史回滚功能，所有 Markdown 都会存储于私有仓库。

- 注意要选择 GitHub、Gitee、GitLab 或 Gitea 作为主要备份方式才可生效，按钮位于页面最下方。

### 自动同步间隔

设置数据自动同步的频率，写作时停止输入后达到设定时间将自动进行备份。

### 主要备份方式

选择 GitHub、Gitee、GitLab 或 Gitea 作为主要备份方式，设置后才可生效。

### 注意事项

- 删除云端同步文件会同时删除远程仓库中的文件
- 请确保访问令牌配置正确且有仓库访问权限
- 建议使用私有仓库保护数据安全
