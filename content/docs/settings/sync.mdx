NoteGen 支持 GitHub、Gitee、GitLab、Gitea、S3 和 WebDAV。选择的平台会成为当前主要同步方式。

![同步配置页，展示 GitHub 仓库、笔记和记录同步选项](https://files.seeusercontent.com/2026/07/16/c1cW/desktop-sync.jpeg)

## 同步对象

| 对象 | 行为 |
| --- | --- |
| Markdown 与资源 | 文件级上传、拉取、删除和编辑停止后的自动同步 |
| 记录、标签、聊天与设置 | 以数据包方式手动或自动上传、下载，并检测跨设备冲突 |

工作区路径、界面缩放等设备特定设置默认排除。开启“不同步敏感配置”时还会排除 Token、密码、图床、MCP 等私密配置。

## 平台配置

### GitHub / Gitee / GitLab / Gitea

填写 Token 和仓库名。GitLab、Gitea 支持自建实例。仓库不存在时，部分平台可由 NoteGen 创建私有仓库。

| 平台 | 已有仓库的最小权限 | 创建仓库时的附加权限 |
| --- | --- | --- |
| GitHub | Fine-grained PAT，Contents 读写 | Administration 读写，并在创建后换成仅授权目标仓库的 Token |
| Gitee | `user_info`、`projects` | 相同 |
| GitLab | `api` | 相同 |
| Gitea | `read:user`、`write:repository` | `write:user`、`write:repository`；旧版可能只能选择 `all` |

- [创建 GitHub Token](https://github.com/settings/personal-access-tokens/new?name=NoteGen&description=NoteGen+sync&expires_in=none&contents=write&administration=write)
- [创建 Gitee Token](https://gitee.com/profile/personal_access_tokens/new)
- [创建 GitLab Token](https://gitlab.com/-/user_settings/personal_access_tokens?name=NoteGen&description=NoteGen+sync&scopes=api)

### S3

配置 Access Key ID、Secret Access Key、Region、Bucket、Endpoint 和可选路径前缀。兼容 AWS S3 及提供 S3 API 的对象存储。先保存并测试连接，再启用自动同步。

### WebDAV

配置服务器 URL、用户名、密码和可选路径前缀。适用于群晖、QNAP、Nextcloud 及其他兼容服务。建议使用 HTTPS。

## 笔记自动同步

可选择关闭或在停止编辑 2 秒至 2 分钟后自动同步。还可启用“打开文件时自动拉取”和“切换文件时自动拉取”。自动拉取可能用远程版本覆盖本地，请只在同步流程稳定后开启。

## 记录与配置自动同步

启用后，记录或设置变更会进入顺序队列并上传。首次启用时请选择：

- 上传本机数据；
- 拉取远程数据；
- 稍后处理。

新设备通常应拉取远程；已有未同步内容的设备应先备份再决定。

## 冲突处理

- **合并两边内容**：保留双方新增记录，同一记录优先保留更新时间较新的版本，并按隐私规则合并设置。
- **拉取远程覆盖本机**：丢弃本机未上传修改。
- **上传本机覆盖远程**：可能丢弃其他设备的新修改。

处理高风险冲突前，建议先从标题栏同步菜单导出本地 ZIP 备份。
