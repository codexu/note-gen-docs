## 配置结构

一个供应商配置保存 Base URL、API Key、代理和请求头，并可包含多个模型。创建配置时可使用 ChatGPT、Gemini、Ollama、LM Studio 等模板，也可以选择自定义供应商。

![模型配置页，展示 NoteGen 免费对话、嵌入和视觉模型](https://files.seeusercontent.com/2026/07/16/jiU6/desktop-models.jpeg)

## 供应商字段

| 字段 | 说明 |
| --- | --- |
| 名称 | 用于识别供应商，建议不要重复 |
| Base URL | OpenAI 兼容接口地址，通常填写到版本号，如 `https://api.openai.com/v1` |
| API Key | 服务密钥；本地服务不校验时可填写任意非空值 |
| 网络代理 | 继承全局代理、直连或使用当前供应商的自定义代理 |
| 自定义请求头 | 添加服务商要求的额外 Header |

不要在 Base URL 后添加 `/chat/completions`、`/embeddings` 等具体端点，NoteGen 会按模型类型拼接请求。

## 模型列表

在同一供应商下添加一个或多个模型。可用类型包括：

- `chat`：对话、Agent、写作与视觉输入；
- `image`、`video`：为相应生成接口预留的类型；
- `tts`：文本转语音；
- `stt`：语音转文本；
- `embedding`：文本向量化；
- `rerank`：知识库结果重排。

模型名称可从供应商接口获取，也可以手动填写。类型必须与接口能力一致。

## 高级参数

- **Temperature / Top P**：通常只调整其中一个。
- **流式响应**：模型支持时实时显示输出。
- **最大输出 Token**：留空时使用供应商默认值。
- **Token 限制参数**：默认 `max_completion_tokens`；旧兼容接口可切换为 `max_tokens`。
- **语音类型**：TTS 模型可填写 `alloy` 等供应商支持的 voice。

保存后运行“检测连接”。连接成功不代表每个模型类型都可用，仍建议分别测试 Chat、嵌入、语音等实际场景。
