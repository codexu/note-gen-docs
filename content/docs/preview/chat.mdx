NoteGen 的对话面板包含两种工作方式：Chat 直接回答和生成内容；Agent 可以规划步骤并调用内置工具、MCP 与 Skills 完成任务。

## Chat

Chat 适合问答、总结、改写、翻译、头脑风暴和根据上下文生成内容。请求可包含：

- 当前输入和历史消息；
- 关联标签中的记录；
- 选中的文件或文件夹；
- 编辑器引用的选区；
- 图片附件；
- 知识库检索片段；
- 与当前问题匹配的记忆和 Skills。

工具栏可配置模型、Prompt、回复语言、关联标签/文件、MCP、知识库、剪贴板监听、新对话和上下文清理。

## Agent

Agent 使用“理解任务 → 制定计划 → 调用工具 → 检查结果”的循环完成多步工作。内置工具覆盖：

- 读取、创建和修改 Markdown；
- 新建、移动和管理文件夹；
- 查询和管理标签、记录与聊天；
- 保存和检索长期记忆；
- 获取系统与工作区信息；
- 调用当前启用的 MCP 工具。

运行过程中可以查看计划、时间线、工具参数和执行结果。失败任务可在保留上下文的情况下继续处理。

## 权限和确认

只读操作和修改操作具有不同风险等级。Agent 在需要时显示确认面板，你可以允许一次、在当前会话允许，或拒绝。删除、覆盖和调用外部工具前应检查目标与参数。

编辑器修改会先显示 Diff。接受后写入文件，拒绝则保留原内容；多文件任务可以在变更面板逐项审阅。

## 上下文管理

- **新对话**：创建独立会话，旧会话保留在历史中。
- **清除上下文**：后续请求不再携带此前消息，但界面历史仍可见。
- **清空对话**：删除当前会话消息。
- **自动摘要**：长对话达到阈值后压缩旧消息，减少 Token 使用。
- **上下文托盘**：查看当前关联的文件、记录、记忆、知识库和工具。

上下文越多不一定越准确。遇到回答混乱、成本过高或旧信息干扰时，减少关联文件、关闭不必要的知识库/MCP，或开始新对话。

## MCP、Skills、知识库与记忆

| 能力 | 作用 |
| --- | --- |
| MCP | 连接外部工具和服务 |
| Skills | 为特定任务提供可复用指令和允许工具 |
| 知识库 | 从工作区向量索引检索相关片段 |
| 记忆 | 跨会话加载偏好或按语义匹配事实 |

这四种能力可以组合使用，但也会扩大模型接收到的数据范围。处理敏感内容前先查看[隐私与数据](/cn/docs/privacy)。

## 消息操作

AI 回复支持复制、朗读、翻译、查看模型信息，以及将内容保存为记录或输出到笔记。具体按钮可随工具栏配置和平台变化。
